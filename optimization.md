# Optimal Samples Selection System — UI 优化指令 v2

> 适用于：CS360/SE360 小组项目最终版本打磨  
> 设计方案：浅色三栏式布局，重构信息架构  
> 基于当前系统截图分析 + 三轮设计迭代

---

## 〇、总体设计方向

### 布局变更：二栏 → 三栏

当前系统采用左右二栏布局（左=输入，右=结果），存在三个问题：
1. 参数输入和样本选择挤在同一栏，信息层级混乱
2. 状态指标（GROUPS / TIME / STATUS）和结果表格共享右栏，视觉权重不清
3. 操作按钮散落在不同位置，无统一操作区

**新布局采用三栏结构**：

```
┌─────────────┬──────────────────────────┬─────────────┐
│  左栏 200px  │      中栏 flex-1         │  右栏 280px  │
│  参数配置     │  样本池 + 结果展示         │  状态面板    │
│  + Solve按钮  │  （核心工作区）            │  + 操作按钮  │
└─────────────┴──────────────────────────┴─────────────┘
```

每栏职责单一：左栏只管"配什么参数"，中栏只管"选什么样本、看什么结果"，右栏只管"跑出了什么、怎么保存"。

### 色彩体系

采用浅色主题，确保投影仪和任何屏幕都可清晰阅读。

## 一、顶栏（Top Bar）

### 现状
无独立顶栏，标题和 tab 混排。

### 改进指令

- 顶栏高度约 48px，白色底 + 底部 1px 边框 `#ebedf2`
- 左侧：品牌标记（28×28px 蓝紫色圆角方块 + 白色图标）+ 应用名称
- 右侧：tab 切换（Computation / Database），使用 pill 样式
  - 未选中 tab：灰色文字 `#8891a5`，无底色
  - 选中 tab：蓝紫色底 `#f0f2ff` + 蓝紫色文字 `#4f6ef7`，圆角 8px

```
┌──[■ logo]  Optimal Samples Selection          [Computation] [Database]──┐
└─────────────────────────────────────────────────────────────────────────┘
```

## 二、左栏 — 参数配置（宽度 200px）

### 现状问题
参数输入框与样本池、操作按钮挤在一起，参数含义和范围不直观。

### 改进指令

**整栏为白色底 `#fff`，右侧 1px 边框分隔，内边距 20px 16px。**

#### 2.1 区块标题样式

所有区块标题统一使用：
```
font: 600 10px/1 sans-serif
color: #8891a5
letter-spacing: 0.1em
text-transform: uppercase
padding: 8px 0 6px
```

标注文字为 "PARAMETERS"。

#### 2.2 参数卡片（M / N / K / J）

每个参数一张独立卡片，纵向堆叠，间距 6px。

**卡片结构**：
```
┌────────────────────┐
│ M total          45 │  ← 左=标签(11px, #8891a5)  右=数值(20px, #1a1f36, monospace)
│ pool size 45–54     │  ← hint(10px, #b0b7c5)
└────────────────────┘
```

**卡片样式**：
- 背景：`#f8f9fc`
- 边框：`1px solid #ebedf2`
- 圆角：10px
- 内边距：10px 12px
- hover 时边框变为 `#c5cad6`

**编辑中状态**（当前正在修改的参数）：
- 边框变为 `#4f6ef7`
- 背景变为 `#f5f6ff`
- 数值颜色变为 `#4f6ef7`

#### 2.3 S 参数卡片（特殊处理）

S 参数不只是一个数字，需要用自然语言解释其含义：

**卡片结构**：
```
┌──────────────────────────┐
│ S coverage               │  ← 标签(11px, #8891a5)
│ cover at least 5 from    │  ← 描述(13px, #4f6ef7, sans-serif, 600)
│ every 5-subset           │
│ threshold 3–7            │  ← hint(10px, #b0b7c5)
└──────────────────────────┘
```

**卡片样式**：
- 背景：`#f0f2ff`（蓝紫浅底，区别于其他参数）
- 边框：`1px solid #d4daff`
- 描述文字随 j 和 s 的值动态更新，例如：
  - j=5, s=5 → "cover at least 5 from every 5-subset"
  - j=4, s=3 → "cover at least 3 from every 4-subset"
  - j=6, s=5 → "cover at least 5 from every 6-subset"

#### 2.4 Solve 按钮

- 位于左栏最底部（`margin-top: auto`，推到底部）
- 蓝紫色填充 `#4f6ef7`，白色文字
- 全宽，高度约 40px，圆角 10px
- 文字："Solve"
- 执行中状态：文字变为 "Solving..."，可加 spinner 动画

## 三、中栏 — 工作区（flex: 1，自适应宽度）

### 现状问题
样本池和结果分布在不同栏，用户视线需要左右跳跃。

### 改进指令

**整栏底色为页面底色 `#f5f6f8`，内边距 20px。**

#### 3.1 样本池区域

**顶部信息行**：
```
Sample pool                        [Random | Manual]  [11 / 45]
```

- 左侧：区块标题 "SAMPLE POOL"
- 中间：模式切换使用嵌套 pill 样式
  - 外壳：`#ebedf2` 底色，圆角 8px，内边距 3px
  - 未选中 pill：透明底，灰色文字 `#8891a5`
  - 选中 pill：白底 + 微阴影 `0 1px 2px rgba(0,0,0,0.06)`，深色文字 `#1a1f36`
- 右侧：选中计数徽章，蓝紫底 `#f0f2ff` + 蓝紫文字 `#4f6ef7`，圆角 6px

**格子网格**：
- 10 列网格（`grid-template-columns: repeat(10, 1fr)`），间距 4px
- 每个格子为正方形（`aspect-ratio: 1`）
- 未选中格子：
  - 白底 `#fff`
  - 边框 `1.5px solid #e6e8ee`
  - 文字颜色 `#c5cad6`（浅灰，可见但不抢眼）
  - 字体：12px monospace 600
- 选中格子：
  - 蓝紫填充底 `#4f6ef7`
  - 边框同色 `#4f6ef7`
  - 白色文字 `#fff`
  - hover 时略深 `#3d5ce5`

**关键设计点**：未选中格子不是隐藏或半透明，而是用浅灰色文字保持可见——用户能看到完整的 1-45 序列，但选中的蓝紫色格子第一时间抓住视线。

#### 3.2 结果区域

区块标题 "RESULT GROUPS"。

**双列卡片网格**：
- `grid-template-columns: repeat(2, 1fr)`，间距 8px
- 每页显示 8 张卡片（4 行 × 2 列）

**单张结果卡片结构**：
```
┌──────────────────────┐
│ #01                  │  ← 组号(11px monospace 700, #8891a5)
│ [12][15][19][27][28][31] │  ← 成员 chip 行
└──────────────────────┘
```

**卡片样式**：
- 白底 `#fff`，边框 `1px solid #ebedf2`，圆角 10px
- 内边距 10px 12px
- hover 时边框变深 `#c5cad6`

**成员 chip 样式**：
- 固定宽度 30px，高度 26px
- 背景 `#f0f2ff`，边框 `1px solid #d4daff`
- 文字 `#3b55c9`，12px monospace 700
- 圆角 6px
- 6 个 chip 一行排列，间距 4px

**分页指示器**（底部居中）：
- 使用圆点式指示器
- 当前页：蓝紫色条 `#4f6ef7`，宽 20px 高 8px，圆角 4px
- 其他页：灰色圆点 `#dfe2e8`，8×8px
- 右侧文字 "1–8 of 32"，11px 浅灰色

## 四、右栏 — 状态面板（宽度 280px）

### 现状问题
GROUPS / TIME / STATUS 三指标与结果表格混排，状态含义不清晰。

### 改进指令

**整栏白底 `#fff`，左侧 1px 边框分隔，内边距 20px 16px。**

#### 4.1 核心数字（Groups Found）

区块标题 "SOLUTION"。

**大数字卡片**：
```
┌──────────────────┐
│                  │
│       32         │  ← 36px monospace 800, #1a1f36, 居中
│   groups found   │  ← 11px sans-serif, #8891a5, 居中
│                  │
└──────────────────┘
```

- 背景 `#f8f9fc`，边框 `1px solid #ebedf2`，圆角 12px
- 内边距 16px，文字居中

#### 4.2 状态徽章（OPTIMAL / FEASIBLE）

**胶囊样式，全宽居中**：

OPTIMAL 状态：
```
┌─────────────────────────┐
│     ●  OPTIMAL          │
└─────────────────────────┘
背景: #e8faf0    边框: 1.5px solid #8fe8b8    文字: #15803d
```

FEASIBLE 状态：
```
┌─────────────────────────┐
│     ●  FEASIBLE         │
└─────────────────────────┘
背景: #fef9ee    边框: 1.5px solid #fad275    文字: #a16207
```

- 圆角 12px，内边距 12px
- 圆点 `●` 为 8×8px 实心圆，颜色与文字同色
- 文字 14px sans-serif 600

#### 4.3 迷你统计行

**双列网格**（`grid-template-columns: 1fr 1fr`，间距 8px）：

```
┌──────────┐ ┌──────────┐
│   12ms   │ │   462    │
│solve time│ │candidates│
└──────────┘ └──────────┘
```

- 背景 `#f8f9fc`，圆角 10px，内边距 12px，文字居中
- 数值：16px monospace 700, `#1a1f36`
- 标签：10px sans-serif, `#b0b7c5`

**solve time 精度规则**：
- time < 1s → 显示毫秒（如 `12ms`）
- time 1s–60s → 显示秒（如 `3.2s`）
- time > 60s → 显示秒（如 `140.7s`）

#### 4.4 运行配置详情（Run Config）

区块标题 "RUN CONFIG"。

**键值对列表**：
```
┌──────────────────────────┐
│ Pool           C(45, 11) │
│ Candidates  C(11,6)=462  │
│ Constraint  C(11,5)=462  │
│ Solver              ILP  │
│ Gap               0.0%   │
└──────────────────────────┘
```

- 背景 `#f8f9fc`，边框 `1px solid #ebedf2`，圆角 10px
- 每行用 `justify-content: space-between`
- 左侧键名：12px sans-serif, `#8891a5`
- 右侧键值：12px monospace 600, `#1a1f36`
- 行间分隔线：`1px solid #f0f2f5`

**设计意图**：演示时老师问"为什么是 32 组"，你可以直接指向这个区域解释组合数和约束规模。

#### 4.5 操作按钮

区块标题 "ACTIONS"。

**按钮排列**：
```
┌──────┐ ┌──────┐
│ Save │ │Export│  ← 并排，各占 50%
└──────┘ └──────┘
┌──────────────┐
│    Clear     │  ← 单独一行，全宽
└──────────────┘
```

- Save 按钮：蓝紫文字 `#4f6ef7`，浅蓝背景 `#f5f6ff`，蓝紫边框 `#c7d0ff`
- Export 按钮：灰色文字 `#64748b`，白底，灰边框 `#ebedf2`
- Clear 按钮：红色文字 `#dc2626`，白底，浅红边框 `#fecaca`
- 所有按钮：圆角 8px，高度约 36px

## 五、Data Base Rounds 界面

### 5.1 文件列表改进

每条记录从单行文件名改为双行展示：

```
┌──────────────────────────────┐
│ 45-16-6-4-4-2-152            │  ← 文件名(monospace, #1a1f36)
│ m=45 n=16 k=6 j=4 s=4 | 152 │  ← 参数解析(11px, #8891a5)
└──────────────────────────────┘
```

- 选中记录：左侧 3px 蓝紫竖条 + 浅蓝背景 `#f0f2ff`
- 未选中记录：hover 时底色变 `#f8f9fc`

### 5.2 POOL 行标注

在 POOL 数字行前添加标签：

```
POOL — 16 samples drawn for this run
[2] [3] [5] [6] [8] [9] [17] [23] [26] [27] [29] [30] [31] [34] [35] [39]
```

- 标签使用区块标题样式（10px uppercase 灰色）
- 数字用灰色 chip（区别于结果的蓝紫 chip）
  - 背景 `#f1f5f9`，边框 `#e2e8f0`，文字 `#475569`

### 5.3 结果组卡片

与 Computation 界面的结果卡片样式完全一致（蓝紫 chip），但改为三列网格：

```
grid-template-columns: repeat(3, 1fr)
gap: 8px
```

- 每张卡片只保留组号 `#01` 和成员 chip，去掉重复的 "Group" 文字标签
- 超过 50 组时使用分页或虚拟滚动

### 5.4 状态面板复用

右侧状态面板结构与 Computation 界面一致，额外显示：
- 文件来源信息
- Load into Computation 按钮（蓝紫色主按钮样式）

## 六、交互细节

### 6.1 参数输入验证

用户输入不合法值时（如 m=30，超出 45–54 范围）：
- 参数卡片边框变红 `#dc2626`
- 背景变浅红 `#fef2f2`
- hint 行显示错误信息（红色文字）："must be 45–54"
- Solve 按钮禁用（降低透明度 + cursor: not-allowed）

### 6.2 Solve 执行状态

点击 Solve 后：
- 按钮文字变为 "Solving..."，可选添加 spinner 动画
- 按钮禁用防止重复点击
- 右栏状态区域可显示进度提示

### 6.3 结果卡片交互

- hover 时边框加深
- 可选：点击某张卡片时，高亮样本池中对应的 6 个数字（加粗边框或背景变化）

### 6.4 键盘快捷键（加分项）

- `Enter` → 触发 Solve
- `Ctrl/Cmd + S` → 触发 Save
- `Esc` → 触发 Clear（需二次确认）
